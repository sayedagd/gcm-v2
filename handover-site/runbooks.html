Operational playbook

Use these steps for backup, restore, release, incident response, and rollback. They are written for a separate frontend and backend deployment model.

Home Overview Backend Schema Environment Print export
  • Backup: use the backend backup service and verify the archive before rotation.
  • Restore: use the restore endpoint with a known-good backup file and confirm schema health afterward.
  • Release: deploy the backend API separately from the frontend static build.
  • Incident response: check schema health, event stream, logs, and auth routes first.
  • Rollback: redeploy the previous frontend build and roll back the backend container/image if migrations were not applied.
