Overview

GCM v2 is an environmental operations platform for companies that collect projects, recyclables, and waste. The core system manages companies, projects, services, trips, drivers, vehicles, suppliers, facilities, inventory, reports, AI sessions, and public landing/store flows.

Frontend

The frontend is a standalone React + Vite application. It uses React Router, Zustand, Mantine, custom utility components, and Tailwind-based styling. Build it separately and point it to the backend API with VITE_API_BASE_URL.

Backend

The backend is an API-only Express service. It owns auth, route modules, migrations, backups, event stream handling, WhatsApp, OCR, and AI services. It does not serve the SPA shell.

Schema

The database contract centers on users, companies, projects, services, trips, logistics tables, system config, and reporting/AI history. Schema alignment was normalized by removing duplicate DDL and aligning runtime contract names.

API

Core route groups include auth, trips, projects, companies, services, vehicles, drivers, facilities, suppliers, and system routes such as schema-health, force-migrate, and WhatsApp QR.

User flows

Admin, operations staff, client, subcontractor, and driver journeys are separated by role so each persona sees an appropriate slice of the platform.

Runbooks

Backup, restore, release, incident response, and rollback steps are documented for the split deployment model. Use schema health first when verifying operational state.
