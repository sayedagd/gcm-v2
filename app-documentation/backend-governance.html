API Versioning

Contract namespace and deprecation policy

  • Stable namespace: /api/v1/*.
  • Compatibility namespace: /api/* aliases to the same handlers.
  • Backward-compatible auth aliases can exist for migration, but clients should target /api/v1/auth/*.
  • Deprecated aliases include headers: Deprecation, Sunset, and Link: ; rel="successor-version".
  • Default sunset date: 2026-12-31, configurable via LEGACY_API_SUNSET.
  • Minimum compatibility window: 6 months before removing deprecated aliases.

Auth Token Policy

Storage and lifecycle controls

  • Access tokens are signed using JWT_SECRET only.
  • Token TTL controlled by JWT_EXPIRES_IN (default 12h).
  • Login sets HttpOnly cookie (AUTH_COOKIE_NAME, default gcm_jwt).
  • Cookie controls: AUTH_COOKIE_SAMESITE and AUTH_COOKIE_SECURE.
  • Logout clears auth cookie explicitly.
  • Expired tokens return 401 with code AUTH_TOKEN_EXPIRED and WWW-Authenticate metadata.
  • Legacy Bearer token response is retained temporarily for compatibility during migration.

Rate-Limit Policy

Endpoint-tier defaults

  • Global API: 1000 / 15m.
  • Auth login: 20 / 15m.
  • Public write (/api/public/contact, /api/public/store): 40 / 15m.
  • SSE token issuance (/api/auth/sse-token): 30 / 15m.
  • AI routes (/api/ai, /api/ai/ocr): 120 / 15m.
  • Sensitive admin ops (/api/system/backup, /api/system/settings, /api/system/force-migrate): 30 / 15m.
  • All tiers can be overridden through RL_* environment variables.

Validation Audit

Write-path sanitization coverage

  • Baseline validation and sanitization middleware is applied to all write methods: POST, PUT, PATCH, DELETE.
  • Body-shape guard enforces object payloads for JSON/urlencoded write requests.
  • Recursive string sanitization is applied globally with URL/file/avatar field exemptions.
  • Route-specific schema and business validation remain active on top of the baseline layer.
  • Middleware entry: backend/src/shared/middleware/requestValidationMiddleware.js.

Maintenance Operations

Backend maintenance script catalog

  • npm run maintenance:create-test-accounts: create/update standard role-based test users.
  • npm run maintenance:inject-services: sync baseline services/facilities seed constants.
  • npm run maintenance:reset-admin: reset primary admin credentials.
  • npm run maintenance:reset-testing: wipe transactional data while preserving service tree and one admin.
  • npm run maintenance:wipe-data: wipe operational data while preserving users/services/config.
  • Archived one-off migration scripts remain under backend/scripts/maintenance/archive for manual, reviewed recovery usage only.
