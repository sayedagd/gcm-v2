Frontend Architecture

Next.js App Router application

The active frontend runtime is frontend-next/ on Next.js 16 + React 19. It uses App Router route groups, middleware-based role protection, server-aware data strategies, and action-based form flows.

  • Entry point: frontend-next/src/app/layout.tsx and route groups under src/app/(*).
  • Route governance: legacy path parity mapped to App Router pages with role-aware middleware.
  • Auth/session boundary: cookie-based session markers and role redirects handled by middleware.ts.
  • Session lifecycle: login sets session expiry cookie and middleware enforces forced re-auth on expiration.
  • Data strategy: public ISR pages + dynamic/no-store portal and internal routes.
  • Route cache governance: explicit static/dynamic/ISR declarations with matrix tracked in milestones.html.
  • React 19 actions: login/logout flows use useActionState + server actions.
  • Loading UX: route-level loading boundaries and deferred split navigation components.
  • Feature decomposition: shared shell/navigation/migration modules are feature-bounded under src/features/*.
  • Feature contract: each feature follows ui, hooks, api, model, tests directory convention.
  • Convention guardrail: npm run check:features enforces folder structure and runs inside lint.
  • Endpoint governance: centralized constants and typed operation wrappers under src/features/core.
  • Bundle governance: npm run ci:perf runs build + bundle budget checks + heavy dependency guardrails.
  • Media delivery: Next image optimization enabled (AVIF/WebP, cache TTL, remote uploads pattern) with lazy-loaded secondary media on landing.
  • Frontend quality gates: Vitest suite covers auth policy/helpers and core API operation wiring (npm run test:run).
  • Split deployment smoke entrypoint: npm run smoke:split (requires deployment URLs via env vars).
Deployment note: deploy frontend-next/ as the frontend project and set NEXT_PUBLIC_API_BASE_URL (and optional server-side API_BASE_URL) to the backend URL.
Legacy runtime: frontend/ is decommissioned. Default scripts are intentionally blocked; only explicit legacy:* scripts remain for emergency comparison.
