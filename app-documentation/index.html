Scope

Why this matters

  • Frontend and backend are hosted as separate Vercel projects.
  • The backend is API-only and no longer serves the SPA shell.
  • API governance is versioned under /api/v1 with OpenAPI-driven typing.
  • Write endpoints pass through global body-shape validation + sanitization middleware.
  • Schema drift is tracked through migration service + OpenAPI schema generation checks.
  • Release confidence includes frontend unit tests, backend OpenAPI contract tests, and split-deployment smoke checks.
  • Runtime observability includes request correlation IDs, structured logs, and threshold-based metrics alerts.
  • Admin-only system routes are centrally protected in backend middleware.
