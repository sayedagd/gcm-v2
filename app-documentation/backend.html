Backend Architecture

API-only Node service

The backend is an API-only Express service with versioned API contracts, standardized errors, object-storage-backed files, and scheduler-safe job execution.

Home Overview Frontend Governance Schema Runbooks Print export
  • Entry point: backend/app.js
  • Database access: backend/database.js
  • Primary pattern: route modules mounted by domain under /api/v1 with legacy alias compatibility.
  • Auth: JWT protect middleware with role guard support
  • Auth lifecycle hardening: configurable token TTL + HttpOnly cookie issuance + explicit logout cookie clear.
  • Error contract: canonical payload via backend/src/shared/utils/apiError.js.
  • Validation baseline: global write-request body-shape guard + recursive payload sanitization middleware.
  • API contract: OpenAPI source at backend/openapi/openapi.v1.json.
  • Rate limiting: endpoint-tier policy for auth/public-write/AI/admin-sensitive operations.
  • Observability: request correlation middleware, structured JSON request logs, and metrics/alert snapshots at /api/v1/system/metrics.
  • Backup/upload storage: S3-compatible object storage with signed URL delivery.
  • Scheduling: in-process cron disabled by default; managed scheduler endpoints protected with idempotency keys + advisory locks.
  • Security and governance policies are consolidated in backend-governance.html.
  • Observability artifacts: backend/src/shared/services/metricsService.js and incident guide in milestones.html.
  • Operational maintenance tools: backend/scripts/maintenance (legacy one-off scripts are archived under backend/scripts/maintenance/archive)
  • Runtime services: migrations, backups, event stream, lazy-loaded WhatsApp/OCR/AI
The backend should be hosted independently from the frontend build and must remain API-only in production.
