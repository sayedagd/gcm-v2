# 04. Full OpenAPI Specification (Reverse-Engineered)

Canonical contract file in repo: `backend/openapi/openapi.v1.json`

This section expands the implemented route surface discovered in route modules and runtime mounts.

## 1. OpenAPI Metadata

- OpenAPI: `3.0.3`
- Title: `GCM Backend API`
- Version: `1.0.0`
- Primary base path: `/api/v1`
- Security: `bearerAuth` (`JWT`)

## 2. Security Model

- `Authorization: Bearer <JWT>` for protected APIs
- HttpOnly cookie auth is also supported server-side (`gcm_jwt` by default)
- Admin-only routes require JWT role = `ADMIN`
- SSE short-lived token endpoint: `GET /api/v1/auth/sse-token`

## 4. Core Request/Response Models

Representative models from contract and code:

- `LoginRequest`: `email`, `password`
- `AuthResponse`: token and user payload
- `ErrorResponse`: canonical envelope with `code`, `error`, `errorEn`, `errorAr`, `traceId`
- `UpsertSuccess`, `DeleteSuccess`, `OperationSuccess`
- Domain entities: `User`, `Company`, `Project`, `Service`, `Facility`, `Trip`

## 5. Authentication Requirements by Prefix

- Public/no auth:
  - `/api/v1/public/*`
  - `/api/v1/ping`
  - `/api/v1/diag`
  - `/api/v1/config` (currently open)
- Protected:
  - most `/api/v1/*` operational and core routes
- Protected + Admin role:
  - `/api/v1/system/*` for backup/health/metrics/schema operations

## 6. Versioning and Compatibility

- Runtime preserves legacy `/api/*` endpoints with deprecation headers and sunset support.
- `backend/vercel.json` rewrites map v1 route aliases into active route handlers.

## 7. Recommended Spec Consolidation

To make this truly machine-complete for SDK generation:

1. Merge all route module signatures into `backend/openapi/openapi.v1.json`.
2. Add path-level auth metadata per route.
3. Add strict schema definitions for trip/notification/store/admin payloads.
4. Add reusable error examples with `traceId`.
