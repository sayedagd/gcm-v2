Trips Module

Trip lifecycle and validation

Trips are the core operational entity. The module defines the data model, lifecycle, document flow, validation rules, and file storage structure used by the backend and UI.

Home Docs archive System knowledge Driver lifecycle Reports Deployment
  • Trip IDs are generated automatically and all active trips require valid driver and vehicle assignments.
  • Manifest, delivery note, and recycle receipt handling is governed by service rules.
  • Validation covers company, project, service, quantity, facility, and document uniqueness checks.
  • Files are stored in a company/project/service/year-month hierarchy.
