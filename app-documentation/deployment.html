Deployment

Build and release workflow

The deployment model is split: deploy the Next.js frontend app and API backend independently, with versioned API contracts and externalized environment configuration.

Home Architecture Docs archive System knowledge Trips Reports Environment
  • Create two Vercel projects only: one rooted at frontend-next/ and one rooted at backend/.
  • Use the architecture reference page to verify service boundaries before first deploy.
  • Set NEXT_PUBLIC_API_BASE_URL in the frontend project to the backend custom domain (https://gcm-back.twision.agency).
  • Set backend env vars in Vercel: DATABASE_URL, JWT_SECRET, CORS_ORIGIN, storage keys, and required AI/integration keys.
  • Keep ENABLE_IN_PROCESS_CRON=false in backend runtime and run scheduled jobs from a managed scheduler hitting authenticated API endpoints.
  • Use /api/v1 URLs for all new clients and automations.
  • Keep app-documentation/ local-only for developer reference; do not deploy it to Vercel.
